Frequently asked questions

How are AWS Availability Zones isolated?
Availability Zones use separate facilities and infrastructure to reduce correlated failures while remaining connected through private, high-bandwidth, low-latency networking.
Are zone names the same for every AWS account?
Zone names such as us-east-1a can map differently between accounts. Use Availability Zone IDs when workloads in multiple accounts must refer to the same physical zone.
Does every AWS service support multi-AZ deployment?
No. Availability Zone support and deployment behavior vary by service and region. Review the service documentation and test the workload's failure behavior before production use.
